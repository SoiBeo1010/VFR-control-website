HOW TO UPDATE:
═════════════════════════════════════════════════════════════════════════

STEP 1: Download the latest ZIP
   - Click the three dots (...) in v0 top right
   - Select "Download ZIP"
   - Extract to a new folder

STEP 2: Stop the current server
   - In VS Code terminal, press Ctrl+C
   - This stops the running python server

STEP 3: Copy the new qc-worker.html
   - From downloaded ZIP: public/qc-worker.html
   - Replace the old one in your current project
   - OR delete the old folder and use the new extracted folder

STEP 4: Restart the server
   - In terminal, run: python main.py
   - Wait for: "INFO:     Uvicorn running on http://0.0.0.0:8000"

STEP 5: Test the new form
   - Open browser: http://localhost:8000
   - Login as QC Worker: qc_worker_1 / password123
   - You'll see the new inspection form

WHY NO DATA SHOWS IN QC MANAGER?
═════════════════════════════════════════════════════════════════════════

POSSIBLE REASONS:

1. ❌ OLD FORM - Data format incompatible
   ✅ FIX: Update to the new qc-worker.html (just downloaded)

2. ❌ API NOT FETCHING - Backend not querying correctly
   ✅ FIX: Check HOTFIX_NEEDED.txt (if it exists)

3. ❌ DATABASE ISSUE - Data not saving properly
   ✅ FIX: Delete qms_system.db, restart server (fresh database)

4. ❌ ROLE PERMISSIONS - QC Manager role not loading data
   ✅ FIX: Check browser console (F12) for errors

   Look for Python error messages

4. Try deleting database and restarting
   - Stop server (Ctrl+C)
   - Delete: qms_system.db file
   - Run: python main.py
   - Test again

5. Contact support with error message
